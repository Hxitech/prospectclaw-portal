Step by Step

6 步从装好带到真能用

按真实使用顺序排列。每一步回答：先做什么、为什么、做到什么算过关。

推荐顺序

安装 → 钳子助手 → 模型与 Gateway → Agent + Skills → 实时聊天验证 → 接渠道

先把钳子助手配好，因为它是最稳定的"第二双眼睛"。后面模型不通、Gateway 崩了、Agent 不回话，都可以先让钳子助手帮你判断从哪一层开始查。

01
安装与第一次启动

先把应用装好，再跑一遍初始设置

桌面包安装完成后，第一次启动会进入"初始设置"。这不是装饰页，是整个系统的起跑线。

  • macOS 首次打开被拦截：拖进"应用程序"，系统设置 → 隐私与安全性 → 仍要打开。
  • Windows 遇到 OpenClaw 安装报错：优先检查管理员权限、npm 缓存和 Git HTTPS 配置。
  • Linux 无桌面环境：建议走 Web 版部署，面板通过 :1420 访问，Gateway 通过 :18789 管理。
过关标准： 初始设置里 Node.js、Git、OpenClaw、配置文件都已通过。

桌面端

Mac / Windows 用户，下载后直接安装，首次启动自动进入初始设置。

Web 版

Linux / 服务器场景。默认 :1420 面板，:18789 Gateway。

02
先把钳子助手配好

先配置钳子助手，不要等出事了才想起它

钳子助手和你的 OpenClaw Agent 不是一回事。它是内置的独立 AI 助手，适合排障、问答、日志分析。

  • 去"钳子助手"页面，把它自己的模型配好。
  • 就算你的 Agent 还没养活，你也已经有一个能看报错、给修复建议的帮手。
  • 后面模型配崩了、Gateway 抽风了、Skill 缺依赖了，都可以先甩给钳子助手分析。
过关标准： 钳子助手能正常对话。你的"修理工"已经上岗。
Assistant Tools

钳子助手内置工具卡片

很多常见操作不用从零描述，点卡片就能触发预设检查流程。不确定点哪张？从"检查系统环境"、"检查 OpenClaw 配置"、"诊断 Gateway"、"一键排障"这四张开始。

🔧

检查 OpenClaw 配置

读取并分析 openclaw.json，帮你看服务商、Gateway 和常见配置错误。

适合：刚装完、刚改完配置、怀疑写错字段时
🏥

诊断 Gateway

检查进程、端口和日志，判断 Gateway 是没启动、没监听，还是状态异常。

适合：实时聊天连不上、端口异常、网关抽风时
💻

检查系统环境

检查 Node.js、npm、OpenClaw CLI 和端口状态，确认底层环境是否完整。

适合：安装卡住、升级后异常、怀疑环境不完整时
🔨

一键排障

配置、模型同步、Gateway、WebSocket、Node.js 串起来做一轮体检。

适合：突然不工作了、不知道先查哪一层时
📋

分析错误日志

从日志里抓 ERROR、WARN、fail 关键词，帮你缩小问题范围。

适合：看得见报错，但一时看不懂时
📦

Skills 管理

查看哪些 Skills 可用、哪些缺依赖，引导安装缺失依赖。

适合：Agent 会聊天但不会干活时
03
配置模型与启动 Gateway

配置模型，然后启动 Gateway

"模型配置"决定 OpenClaw 用什么模型，"服务管理"决定 Gateway 是否正常跑。两个加起来才算把主体点亮。

  • 至少 1 个主模型，建议 1-2 个备选。
  • 模型测试通过后保存，必要时重启 Gateway 让配置生效。
  • "服务管理"看 Gateway 是否绿色运行，出问题优先看日志。
v1.2.8 Gateway 稳定性增强

仪表盘刷新节流（5 秒）、TCP 连接重试、停止检测阈值 2→3 次、自动重启前增加 3 秒延迟。如果之前遇到 Gateway 频繁闪断，升级到 v1.2.8 会显著改善。

过关标准： 主模型可用，Gateway 绿色运行中。
04
创建 Agent 并挂上 Skills

先创建 Agent，再给它挂上能干活的 Skills

打开"Agent 管理"，先做出一只职责单一、名字清楚、模型明确的 Agent。然后打开"Skills"页面确认关键能力可用。

  • 第一次建议单一职责 Agent，不要一开始就堆太多设定。
  • Agent ID 建议全小写、简单好记；名称可以自然一点。
  • 模型决定它聪不聪明，Skills 决定它会不会干活，身份文件决定它像不像你要的角色。
v1.2.8 SkillHub 技能商店

Skills 页面新增"商店"标签页，可在线浏览、搜索、一键安装 Agent 技能。替代旧版 CLI 命令，无需手动操作命令行。支持按 Agent 分配不同 Skills 目录。

过关标准： Agent 列表有一只叫得出名字的 Agent，至少一个关键 Skill 可用。

常见误区

"页面里能看到 Skill，所以应该已经能用。" —— 不一定，要看它是否显示为可用状态。

建议

先保证 1-2 个关键能力真的能跑，再扩张技能树。

05
实时聊天验证

去实时聊天，让它开口

打开"实时聊天"发第一条消息。很多配置在页面上"像是好了"，但只有这里回得出来，才说明整条链路真的通了。

回得出来意味着：

  • Gateway 在工作。
  • 模型能正常调用。
  • Agent 已被正确加载。
  • 权限没有拦截。
v1.2.8 Chat OpenClaw 4.5+ 兼容

支持 Agent 事件流——生命周期、计划、审批、思考、命令输出事件。打字指示器显示实时计时，3 分钟超时保护避免静默无回复。

第一次测试可以这样问
你好，请用三句话介绍你是谁、你能做什么、你现在用的是哪个模型。
过关标准： 收到第一条稳定回复。从"安装成功"升级到"可以开工"。
06
接消息渠道（可选）

先明白规律，再接飞书和 Telegram

"消息渠道"页面统一管理外部接入。无论接什么平台，两个前提一样：Gateway 先运行，保存后做平台侧发布或配对审批。

  • 飞书 / Lark：创建企业自建应用，开启机器人能力，拿到 App ID / App Secret，事件订阅建议用 WebSocket 长连接，保存后做配对审批。
  • Telegram：通过 @BotFather 创建 Bot，拿到 Token，再准备允许的用户 ID，填到渠道配置保存即可。
  • 保存后不生效？优先检查 Gateway 是否在运行、平台侧应用是否已发布、机器人是否已添加到群中。
过关标准： 机器人能在对应平台收到并回复消息，或已完成凭证校验和配对。

飞书 / Lark

需要 App ID、App Secret，支持内置/官方插件两种模式。发布应用版本后机器人对他人可见。

Telegram

需要 Bot Token 和允许的用户 ID。最轻量，适合最快跑通第一条外部渠道。

一个很好用的习惯

每完成一步，不确定算不算"过关"，直接问钳子助手："我刚完成了模型配置，下一步应该做什么？" 比在页面里来回猜快很多。

Daily Ops

日常管理，先记住这 4 件事

用顺以后，真正省时间的不是"会配"，而是"知道该盯哪里"。

01

模型要稳

Agent 突然变木，大多数时候是模型掉线。先看 API Key、Base URL、主模型连通性，再看备选模型能不能顶上。

02

Gateway 要关好门

只自己用时选"仅本机使用"。要共享就配好 Token 或密码。把门关好比把门打开重要。

03

Agent 要定期整理

名字和职责写清楚，大改前先备份，一个 Agent 只承担一类核心任务。出问题时你知道该修谁。

04

先看日志再找助手

排障顺序：实时聊天表现 → 日志查看 → 系统诊断 → 钳子助手。别一出错就重装。

日常高频页面

钳子助手：问路、排障、卡片触发。服务管理：启停 Gateway，配置备份。日志查看：Gateway、错误、守护进程日志。记忆文件：按 Agent 查看工作记忆。关于：看推荐稳定版，版本切换。

Troubleshooting

排障清单

最常见的坑变成固定检查动作。

Agent 不回话

  • 实时聊天是否连上 Gateway？
  • 模型配置里主模型测试通过？
  • 目标 Agent 绑定了模型？
  • Gateway 认证没有拦截请求？
  • 把报错发给钳子助手判断层级。

能聊天但不会用技能

  • Skills 页面对应 Skill 是否可用状态？
  • 是否缺命令行依赖或环境变量？
  • Gateway 工具权限是否设得太严？
  • Agent 规则有没有卡死能力边界？
  • 把状态贴给钳子助手，通常它能直接告诉你先补哪个依赖。

突然一堆报错

  • 最近是否切了 OpenClaw 版本？
  • 当前版本是否高于推荐稳定版？
  • 最近有没有改服务商、Token 或安全设置？
  • 先确认哪个环节先变了，再修。
  • 截图或粘日志给钳子助手缩小排查范围。

Agent 养死了怎么抢救

  • 先别删 Agent、重装或全量重配。
  • 整理三句话：之前能做什么、现在哪里不对、最近改过什么。
  • 丢给钳子助手：是缺氧、断粮还是网关翻车了？
  • 按顺序查：环境 → 模型 → Gateway → Skills → Agent 本身。
可以直接复制给钳子助手的话
我在 ProspectClaw 里的 OpenClaw Agent 之前能正常回复，现在不行了。请按环境、模型、Gateway、Skills、Agent 配置几个层级，帮我判断最可能的原因并告诉我先去哪个页面检查。
Glossary

9 个基本概念速查

说人话版本。

ProspectClaw ≠ OpenClaw

ProspectClaw 是管理面板（驾驶舱），OpenClaw 是底层引擎（发动机）。

Gateway 是总入口

聊天、模型调用、Agent 行为都经过它。"AI 不聪明"很多时候是入口没通。

Agent 是角色不是窗口

有名字、模型、工作区、规则、记忆和技能边界。新建 Agent = 造一个新角色。

Skill 是能力插件

决定 Agent 除了聊天还能干什么。有些 Skill 还依赖命令行工具或环境变量。

SkillHub 是技能商店

v1.2.8 新增，在线浏览、搜索、一键安装 Skills。替代旧版 CLI 命令。

钳子助手 ≠ 你的 Agent

钳子助手是内置教练兼修理工，Agent 才是你真正要养活的那只虾。

主模型和备选都重要

只配一个也能跑，但一抽风就会带着 Agent 一起抽风。建议至少配一个备选。

工作区是 Agent 的生活区

身份定义、行为规则、记忆文件、任务资料都在这。类比员工的工位和文件夹。

版本不是越新越香

推荐稳定版适合长期使用。升得太快可能先收获兼容性惊喜。
