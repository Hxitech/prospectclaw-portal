Step by Step

6 步把系统从"装好了"变成"能干活"

按实际使用顺序排的。每步说清做什么、为什么、怎么算过关。

推荐顺序

安装 → 钳子助手 → 模型与 Gateway → Agent + Skills → 实时聊天验证 → 接渠道

为什么先配钳子助手？因为后面不管哪个环节出问题，你都可以直接问它。相当于还没上战场就先把军医找好了。

这 6 步会让你完成
安装并首次启动 ProspectClaw
配好钳子助手当排障帮手
配模型、把 Gateway 跑起来
建一只 Agent 并挂上 Skills
实时聊天验证整条链路
接飞书 / Telegram 等渠道
01
安装与第一次启动

装好之后先过一遍初始设置

装好打开后会进"初始设置"页面，别跳过，把每项检查过一遍。这些是后面所有功能的地基。

  • macOS 第一次打开被拦：拖进应用程序，去系统设置 → 隐私与安全性 → 仍要打开。
  • Windows 遇到 OpenClaw 安装报错：检查管理员权限、npm 缓存和 Git HTTPS 配置。
  • Linux 没有桌面环境：走 Web 版部署，面板 :1420，Gateway :18789。
怎么知道搞定了： 初始设置里 Node.js、Git、OpenClaw、配置文件全部通过。

桌面端

Mac / Windows 用户，下载安装后首次启动自动进初始设置。

Web 版

Linux / 服务器场景。默认 :1420 面板，:18789 Gateway。

02
配钳子助手

先有个能帮你看问题的东西

钳子助手跟你后面要养的 OpenClaw Agent 不是一回事。它是内置的独立 AI 助手，专门用来排障、问答、看日志。

  • 去"钳子助手"页面，把它的模型配好。
  • 就算你的 Agent 还没弄好，你已经有了一个能看报错、给修复建议的帮手。
  • 后面模型出问题了、Gateway 挂了、Skill 缺依赖了，都可以甩给它。
怎么知道搞定了： 钳子助手能正常聊天。你的"技术客服"已经上岗了。
Assistant Tools

钳子助手内置的快捷卡片

常见操作不用从零描述，点卡片就行。不知道点哪张？从"检查系统环境"和"一键排障"开始。

🔧

检查 OpenClaw 配置

读取 openclaw.json，帮你看服务商、Gateway 和常见配置写错的地方。

什么时候用：刚装完、改完配置、怀疑哪里写错了
🏥

诊断 Gateway

检查进程、端口和日志，看看 Gateway 是没启动、没监听，还是状态不对。

什么时候用：聊天连不上、端口报错、Gateway 反复抽风
💻

检查系统环境

看 Node.js、npm、OpenClaw CLI 和端口状态，确认底层环境完整。

什么时候用：安装卡住、升级之后突然不对劲
🔨

一键排障

配置、模型、Gateway、WebSocket、Node.js 全部串起来跑一遍体检。

什么时候用：突然不工作了、不知道先查哪里
📋

分析错误日志

从日志里捞 ERROR、WARN、fail 这些关键词，帮你缩小问题范围。

什么时候用：看得见报错但看不懂什么意思
📦

Skills 管理

看看哪些 Skills 能用、哪些缺依赖，引导你把缺的补上。

什么时候用：Agent 能聊天但不会干活
03
配模型、启动 Gateway

模型决定 AI 多聪明，Gateway 是总开关

"模型配置"管的是 OpenClaw 用哪个 AI 模型，"服务管理"管的是 Gateway 跑没跑。两个都搞定了，系统主体才算亮起来。

  • 至少配 1 个主模型，最好再加 1-2 个备选，主的挂了还能顶上。
  • 模型测试通过后保存。如果改了配置，可能要重启 Gateway 才生效。
  • 去"服务管理"确认 Gateway 是绿色运行中，不是的话看看日志说了啥。
v1.2.8 Gateway 更稳了

之前有些人碰到 Gateway 频繁闪断，1.2.8 改进了好几个地方：刷新不再那么频繁、连接断了会自动重试、重启前会多等几秒避免打架。升到 1.2.8 这类问题会少很多。

怎么知道搞定了： 主模型测试通过，Gateway 绿色运行中。
04
建 Agent、挂 Skills

先建一只简单的，别一上来堆太多

去"Agent 管理"建你的第一只 Agent。名字取清楚、职责写明白、模型选好，先跑起来再说。

  • 第一只 Agent 建议只干一件事，别把所有功能都塞给它。
  • Agent ID 全小写、简短好记就行；名字随意。
  • 模型管它聪不聪明，Skills 管它会不会干活，身份文件管它像不像你要的角色。
v1.2.8 有了 SkillHub 技能商店

Skills 页面多了个"商店"标签页，在线搜技能、点一下就装好，不用再敲命令行了。不同 Agent 还能分配不同的 Skills 目录。

怎么知道搞定了： Agent 列表里有一只你建的 Agent，至少一个关键 Skill 是可用状态。

容易踩的坑

"页面里能看到 Skill 所以应该能用了吧？"——不一定，要看它是不是显示"可用"状态。

建议

先确保 1-2 个核心能力真的能跑，再慢慢加。

05
实时聊天验证

发条消息，看看它到底能不能回

去"实时聊天"发一条消息。前面配的那些东西，在各自页面上可能"看着像好了"，但只有这里回得出来，才说明整条链路真的通了。

回得出来意味着：Gateway 在跑、模型能调、Agent 加载成功、权限没拦。

v1.2.8 聊天兼容 OpenClaw 4.5+

现在能看到 Agent 在干嘛了——思考中、在规划、等审批这些状态都会显示出来。发消息后底部会有个计时器，超过 3 分钟没回会自动提醒，不会再出现发了消息石沉大海的情况。

第一次可以这样问
你好，请用三句话介绍你是谁、你能做什么、你现在用的是哪个模型。
怎么知道搞定了： 收到了 Agent 的回复。从"装好了"正式升级到"能干活了"。
06
接消息渠道（可选）

想让 Agent 出现在飞书或 Telegram 里

"消息渠道"页面统一管外部接入。不管接什么平台，两个前提都一样：Gateway 先跑起来，保存后去平台那边做发布或审批。

  • 飞书 / Lark：需要在飞书开放平台创建企业自建应用，开启机器人能力，拿到 App ID 和 App Secret。事件订阅建议用 WebSocket 长连接。保存之后还要去飞书那边发布应用版本。
  • Telegram：通过 @BotFather 创建 Bot 拿到 Token，准备好允许的用户 ID，填进来保存就行。最简单的外部渠道。
  • 保存后不生效？先检查 Gateway 是不是在跑、平台那边的应用是不是已发布、机器人是不是已经加到群里了。
怎么知道搞定了： 机器人在飞书或 Telegram 里能收到消息并回复。

飞书 / Lark

需要 App ID 和 App Secret，支持内置/官方插件两种模式。发布后机器人才对他人可见。

Telegram

需要 Bot Token 和用户 ID。最轻量的选择，适合最快跑通第一个外部渠道。

养成一个好习惯

每做完一步，不确定算不算搞定了，直接问钳子助手："我刚配好了模型，下一步该干嘛？" 比自己在页面里来回找快多了。

Daily Ops

日常使用，盯住这 4 件事就够了

用熟了之后，真正省时间的不是"会配"，是"知道该看哪"。

01

模型别断

Agent 突然不理人，十有八九是模型掉了。先去检查 API Key、Base URL、主模型连通性。配了备选模型的话，看备选能不能顶上。

02

Gateway 别敞开

自己用就选"仅本机"，要共享就配好 Token 或密码。安全设置没事别乱改。

03

Agent 别乱堆

名字和职责写清楚，改大的之前先备份，一只 Agent 就干一类活。出问题的时候你能马上知道该去改谁。

04

先看日志再求助

出问题了别急着重装。先看看实时聊天的表现，再看日志，然后跑一遍系统诊断，最后实在搞不定了再甩给钳子助手。

日常最常用的几个页面

钳子助手：问问题、排障。服务管理：启停 Gateway、备份配置。日志查看：看 Gateway 和错误日志。记忆文件：看 Agent 的工作记忆。关于：看当前版本。

Troubleshooting

排障清单

最常踩的坑，按症状来查。

Agent 不回话

  • 实时聊天是不是连上 Gateway 了？
  • 模型配置里的主模型测试能通过吗？
  • Agent 绑了模型没有？
  • Gateway 的认证设置有没有拦请求？
  • 把具体报错发给钳子助手，让它帮你判断卡在哪一层。

能聊天但不会用技能

  • 去 Skills 页面看看对应 Skill 是不是"可用"状态。
  • 有没有缺命令行工具或环境变量？
  • Gateway 的工具权限是不是设太严了？
  • Agent 的规则有没有把某些能力限死？
  • 截个状态图给钳子助手，它一般能直接告诉你缺什么。

突然一堆报错

  • 最近是不是切了 OpenClaw 版本？
  • 当前版本比推荐稳定版高吗？
  • 最近有没有改过服务商、Token 或安全设置？
  • 先搞清楚哪个东西先变了，再修。
  • 截图或者复制日志给钳子助手。

Agent 彻底不工作了

  • 先别删 Agent，也别重装。
  • 想三个问题：之前能干嘛、现在哪不对、最近改了什么。
  • 把这三句话甩给钳子助手。
  • 按这个顺序查：环境 → 模型 → Gateway → Skills → Agent 本身。
直接复制这段话给钳子助手
我在 ProspectClaw 里的 OpenClaw Agent 之前能正常回复，现在不行了。请按环境、模型、Gateway、Skills、Agent 配置几个层级，帮我判断最可能的原因并告诉我先去哪个页面检查。
Glossary

9 个概念，用大白话说一遍

看完这些就不会被术语绕晕了。

ProspectClaw 和 OpenClaw 什么关系

ProspectClaw 是你看到的这个软件界面（驾驶舱），OpenClaw 是在后台干活的引擎。没有 OpenClaw，ProspectClaw 就是个空壳。

Gateway 是什么

你可以理解为系统的总调度。聊天消息、模型调用、Agent 行为全走它。Gateway 不跑，什么都用不了。

Agent 是什么

Agent 是一个"角色"。它有自己的名字、模型、规则、记忆和技能范围。新建 Agent 就是造一个新角色出来。

Skill 是什么

决定 Agent 除了聊天还能干什么的插件。有些 Skill 还需要额外装命令行工具或配环境变量。

SkillHub 是什么

v1.2.8 新加的技能商店。以前装 Skill 要敲命令，现在在界面里搜一下点安装就行。

钳子助手和 Agent 什么区别

钳子助手是软件内置的帮你排障的工具，Agent 是你要创建和训练的 AI 角色。一个是客服，一个是你的员工。

为什么要配备选模型

只配一个也能跑，但 AI 服务偶尔会抽风。有个备选的话，主的挂了还能自动切过去，Agent 不会跟着一起挂。

工作区是什么

Agent 的"工位"。身份定义、行为规则、记忆文件、任务资料都放在这。相当于这个员工的文件夹。

版本要不要追最新的

不建议。推荐用"稳定版"就行。升太快可能碰到兼容问题，到时候排障比省的时间还多。
